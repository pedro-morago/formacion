# Fundamentos de QA

Los conceptos base sobre los que se construye todo lo demás. Si es tu primera vez por aquí, este es el orden recomendado:

1. [¿Qué es QA?](/es/fundamentals/what-is-qa) — qué significa realmente asegurar calidad, y la diferencia entre QA, QC y testing.
2. [Tipos de testing](/es/fundamentals/types-of-testing) — funcional, no funcional, caja negra/blanca, regresión, smoke…
3. [La pirámide de testing](/es/fundamentals/the-testing-pyramid) — cómo distribuir el esfuerzo entre tests unitarios, de integración y E2E.
4. [Diseño de casos de prueba](/es/fundamentals/test-case-design) — técnicas para elegir *qué* probar: particiones de equivalencia, valores límite, tablas de decisión.
5. [Reporte de bugs](/es/fundamentals/bug-reporting) — cómo escribir un bug que un desarrollador pueda reproducir y arreglar a la primera.
6. [La evolución del rol de QA](/es/fundamentals/the-evolving-qa-role) — qué es calidad, por qué el testing manual no escala, y hacia dónde va el rol.
