# Estrategia y procesos

Cómo se organiza el testing más allá del test individual: la estrategia ágil, decidir qué se prueba en qué capa, y los procesos de calidad alrededor del desarrollo.

- [Capas de testing en backend](/es/strategy/backend-testing-layers) — unit / integración / E2E de API, con ejemplos.
- [Capas de testing en frontend](/es/strategy/frontend-testing-layers) — unit / component / E2E, y quién escribe qué.

- [Testing de microservicios](/es/strategy/microservices-testing) — aislado vs integrado, contract testing y herramientas.
