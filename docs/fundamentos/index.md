# Fundamentos de QA

Los conceptos base sobre los que se construye todo lo demás. Si es tu primera vez por aquí, este es el orden recomendado:

1. [¿Qué es QA?](/fundamentos/que-es-qa) — qué significa realmente asegurar calidad, y la diferencia entre QA, QC y testing.
2. [Tipos de testing](/fundamentos/tipos-de-testing) — funcional, no funcional, caja negra/blanca, regresión, smoke…
3. [La pirámide de testing](/fundamentos/piramide-de-testing) — cómo distribuir el esfuerzo entre tests unitarios, de integración y E2E.
4. [Diseño de casos de prueba](/fundamentos/diseno-de-casos-de-prueba) — técnicas para elegir *qué* probar: particiones de equivalencia, valores límite, tablas de decisión.
5. [Reporte de bugs](/fundamentos/reporte-de-bugs) — cómo escribir un bug que un desarrollador pueda reproducir y arreglar a la primera.
6. [La evolución del rol de QA](/fundamentos/evolucion-del-rol-de-qa) — qué es calidad, por qué el testing manual no escala, y hacia dónde va el rol.
